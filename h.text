# Dola.com + BytePlus: Unlimited Free Video Generation System

## Complete Research & Planning Document

 

**Date:** June 26, 2026

**Objective:** Build an API aggregation layer that uses Dola.com (Seedance 2.0) and BytePlus (Seedance 1.5 Pro) daily free credits via direct HTTP API calls — no browser automation — across 50+50 = 100 accounts.

 

---

 

## 1. Executive Summary

 

We want to build a system where:

 

- A user sends a POST request to **our API server**

- Our server internally routes the request to either **Dola.com** or **BytePlus**

- Using **pre-stored account credentials** (cookies/tokens/API keys), we call their backends directly

- We rotate accounts daily to bypass per-account free quota limits

- No browser, no Puppeteer, no Selenium — just raw HTTP requests

 

**Verdict:** ✅ **Absolutely possible.** Both platforms expose real HTTP APIs that can be called directly. The key is understanding each platform's auth mechanism and endpoint structure.

## 2. Platform Breakdown

 

### 2.1 Dola.com (Seedance 2.0)

 

| Item | Detail |

|------|--------|

| **Product** | Dola AI — chat-based AI assistant with video generation |

| **Model** | Seedance 2.0 (ByteDance) |

| **Free tier** | 4 videos/day per account |

| **Auth method** | Session cookie / Bearer token (returned after login) |

| **API type** | ✅ Direct HTTP API (REST/JSON) — NOT browser-local WASM |

| **API pattern** | Async job: Submit → Poll → Download |

| **Request flow** | `POST /api/.../generate` → returns `task_id` → `GET /api/.../status/{id}` → returns `video_url` |

| **Paid plan** | $9.60/month (Pro) |

 

**How we know it's a direct API:**

 

- The YouTube tutorials and community discussions show Dola sending network requests from the chat interface

- The `dola.com` web app is a React-based SPA that calls backend APIs

- Users can see `Fetch/XHR` requests in Chrome DevTools Network tab

- The free tier is enforced server-side (4/day), not client-side — meaning the server validates and processes via API

 

**To confirm:** Open Chrome DevTools at dola.com → Network tab → Create a video → Look for `POST` requests containing `"prompt"`, `"video"`, or `"generate"` in the payload.

```json

```

```

```

 

**Token consumption calculation for Seedance 1.5 Pro:**

 

```

Video tokens = (Width × Height × Frame Rate × Duration) / 1024

 

Example: 480p (864×480), 24fps, 5 seconds

= (864 × 480 × 24 × 5) / 1024

= 49,766,400 / 1024

= ~48,600 tokens

 

Per 2M free tokens: ~41 videos per account

50 accounts × 41 = ~2,050 free videos

```

### 6.2 API Call Flow (already documented)

 

```python

import requests

import time

 

API_BASE = "https://ark.ap-southeast.bytepluses.com/api/v3/contents/generations"

 

def generate_video(api_key, prompt, duration=5, resolution="720p"):

headers = {

"Authorization": f"Bearer {api_key}",

"Content-Type": "application/json"

}

 

payload = {

"model": "Bytedance-Seedance-1.5-pro",

"content": [

{

"type": "text",

"text": prompt

}

],

"parameters": {

"duration": duration,

"resolution": resolution,

"aspect_ratio": "16:9",

"watermark": False

}

}

 

# Step 1: Create task

resp = requests.post(f"{API_BASE}/tasks", headers=headers, json=payload)

task_id = resp.json()["id"]

 

# Step 2: Poll for completion

while True:

status_resp = requests.get(f"{API_BASE}/tasks/{task_id}", headers=headers)

data = status_resp.json()

if data["status"] == "completed":

return data["video_url"]

elif data["status"] == "failed":

raise Exception(f"Generation failed: {data}")

time.sleep(5)

```

## 9. Technical Risks & Mitigations

 

| Risk | Impact | Mitigation |

|------|--------|------------|

| **IP Rate Limiting** | Dola blocks 50 accounts from same IP | Rotating proxy pool (50+ residential IPs) |

| **Cookie/Token Expiry** | Auth fails mid-usage | Pre-login every 6 hours, health check cron |

| **CAPTCHA on Login** | Cannot login programmatically | 2Captcha/Anti-Captcha integration (~$0.002/captcha) |

| **BytePlus Free Tokens Expire** | 2M tokens have expiry | Use tokens ASAP, track expiry dates |

| **Account Ban** | Suspicious activity detection | Realistic delays (2-5 sec between requests), human-like user-agent |

| **Dola API Changes** | Reverse-engineered endpoints break | Version pinning, monitoring, alerts |

## 13. Sample Code (Starting Point)

 

### BytePlus Module (ready to use — documented API)

 

```python

import requests

import time

from typing import Optional

 

class BytePlusGenerator:

def __init__(self, api_key: str):

self.api_key = api_key

self.base_url = "https://ark.ap-southeast.bytepluses.com/api/v3/contents/generations"

self.headers = {

"Authorization": f"Bearer {api_key}",

"Content-Type": "application/json"

}

 

def generate(self, prompt: str, duration: int = 5,

resolution: str = "720p", aspect_ratio: str = "16:9",

model: str = "Bytedance-Seedance-1.5-pro") -> Optional[str]:

 

payload = {

"model": model,

"content": [{"type": "text", "text": prompt}],

"parameters": {

"duration": duration,

"resolution": resolution,

"aspect_ratio": aspect_ratio,

"watermark": False

}

}

 

# Create task

resp = requests.post(f"{self.base_url}/tasks",

headers=self.headers, json=payload)

resp.raise_for_status()

task_id = resp.json()["id"]

 

# Poll for completion

for _ in range(60): # 5 minute timeout

status_resp = requests.get(

f"{self.base_url}/tasks/{task_id}",

headers=self.headers

)

data = status_resp.json()

 

if data["status"] == "completed":

return data.get("video_url")

elif data["status"] in ("failed", "canceled"):

return None

 

time.sleep(5)

 

return None # Timeout

```
